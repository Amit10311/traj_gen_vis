83 ## 3.0 Common procedure - map and target trajectory
84 
85 The chasing algorithm receives 1) prior map (in the form of octomap) and 2) The future trajectory of target in either fully known([one shot mode](#oneshot)) or partially updated way. The environments provided are still to be updated more. Sorry for my laziness ...
86 
87 *1) Map*
88 
89  In "worlds" folder, you can find *.bt files (e.g. map3.bt which appears in the figures of this page). You can build your own map by following the [instruction](https://github.com/ethz-asl/rotors_simulator/wiki/Generate-an-octomap-from-your-world) of rotors simulator wiki
90 
91 *2) target trajectory*
92 
93  In "data/${map name}" folder, you can find path*.txt files which can be loaded in the gui of the package. You may also generate your own path by traj_gen package. For saving and loading a target path, [this page](https://github.com/icsl-Jeon/traj_gen) is referred(traj_gen).

95 ## 3.1 Simulation without gazebo
96 
97 This mode does not require gazebo. It only tests the proposed chasing policy and display the result in Rviz. Please run the following command:
98 
99 ```
100 roslaunch auto_chaser simulation_without_gazebo.launch
101 ```
102 
103 ### Step 1 : Spawning chaser from user selection
104 
105 This step should precede the following steps. In rviz tool properties widget, set the topic name of 2D Nav Goal as /chaser_init_pose. If you don't remember the topic, just click the ```set chaser pose button``` in gui in the next time :). If things done, move on to one of the following two modes : one-shot or receding horizon method
106 
107 ### Step 2 - option A : One-shot mode (offline trajectory computation)
108 
109 <img src= "https://github.com/icsl-Jeon/traj_gen_vis/blob/master/img/tutorial1.gif">
110 
111 For this one-shot mode, the chaser is assumed to have the full information for the future trajectory of target. Based on the future path, the trajectory will be computed immediately.
112 
113 
114 
115 ### Step 2 - option B : Receding horizon mode (online trajectory computation)
116 
117 <img src= "https://github.com/icsl-Jeon/traj_gen_vis/blob/master/img/tutorial2.gif">
118 
119 For the receding horizon method, the chaser does not have full information of the future movement of target. Instead, the future movement of target for only short horizon (**"pred_horizon"** parameter in launch files) with a regular interval (**pred_horizon** - **early_end_time** . Please check it in launch files!) is fed to chaser and chaser should keep re-plan in response to the updates.

121 ## 3.2 Simulation with gazebo
122 
123 This package provides a simulation environment by employing package "rotors_simulator". There, the chaser is equipped with vision-sensor (vi-sensor) where you can check the actual capture of the target. In contrast to the [*without gazebo* mode](#without), the initial spawn pose of the chaser should be set with the arguments **chaser_x** , **chaser_y**.
124 
125 ```
126 roslaunch auto_chaser simulation_with_gazebo.launch
127 ```
128 
129 The remaining procedure is the same with the section 3.1.
130

143  <param name="robot_description" command="$(arg urdf_file)" />
144 
145  <!-- Gazebo model spawner -->
146  <node name="spawn_turtlebot_model" pkg="gazebo_ros" type="spawn_model"
